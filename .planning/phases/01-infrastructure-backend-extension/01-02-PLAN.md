    **If any tests fail:** Investigate whether the failure is caused by the COOP/COEP headers or Transformers.js installation. Common failure modes:
    - If COEP `credentialless` causes proxy issues with `/api-proxy` -> localhost:3000: Per D-06, add a `Cross-Origin-Resource-Policy: cross-origin` header to the Vite proxy configuration. Do NOT switch to `require-corp`. The fix in `frontend/vite.config.ts` proxy block would be:
      ```typescript
      proxy: {
        '/api-proxy': {
          target: 'http://localhost:3000',
          changeOrigin: true,
          rewrite: (path: string) => path.replace(/^\/api-proxy/, ''),
          configure: (proxy) => {
            proxy.on('proxyRes', (proxyRes) => {
              proxyRes.headers['cross-origin-resource-policy'] = 'cross-origin';
            });
          },
        },
      },
      ```
    - If HMR WebSocket fails: This should not happen with `credentialless` (only happens with `require-corp`). If it does, check browser console for CORP-related blocking messages.
