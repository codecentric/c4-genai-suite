       **Deviation note (INFRA-01):** REQUIREMENTS.md lists `assetsInclude` as a Vite config item. Transformers.js v4 with onnxruntime-web 1.26+ loads WASM files via native `fetch()` at runtime rather than Vite asset pipeline imports, so `assetsInclude: [/\.wasm$/]` is unnecessary and omitted. The `worker.format: 'es'` setting replaces the Worker bundling concern that `assetsInclude` was originally intended to address.

       headers: {

       Per D-04: headers are set ONLY in Vite dev server, not production config.
